## Project Overview

Capacitor plugin (`smartwinnr-capacitor-daily`) that wraps the Daily.co video calling SDK for use in Ionic apps. Provides native iOS and Android implementations plus a web stub. Targets Capacitor 7.x.

## Build & Development Commands

```shell
npm run build          # Clean, generate docs, compile TS, bundle with Rollup
npm run watch          # TypeScript watch mode (no Rollup/docgen)
npm run verify         # Build + validate all platforms (iOS, Android, web)
npm run verify:ios     # xcodebuild for iOS
npm run verify:android # Gradle clean build + test for Android
npm run lint           # ESLint + Prettier + SwiftLint (check only)
npm run fmt            # ESLint + Prettier + SwiftLint (auto-fix)
npm run docgen         # Regenerate API docs in README.md from JSDoc in definitions.ts
```

Android tests: `cd android && ./gradlew test` (unit) or `./gradlew connectedAndroidTest` (instrumented).

## Architecture

The plugin follows the standard Capacitor plugin pattern with three layers:

**TypeScript layer** (`src/`):
- `definitions.ts` — Plugin interface (`SmartWinnrDailyPlugin`) defining `joinCall`, `endCall`, `echo`, and event listeners (`onJoined`, `onLeft`)
- `index.ts` — Registers the plugin with Capacitor via `registerPlugin`, lazy-loads web implementation
- `web.ts` — `SmartWinnrDailyWeb` stub (no-op implementations for browser)

**iOS layer** (`ios/Sources/SmartWinnrDailyPlugin/`):
- `SmartWinnrDailyPlugin.swift` — Capacitor `@objc` bridge exposing methods to JS
- `SmartWinnrDaily.swift` — Core logic
- `DailyCallViewController.swift` — UIViewController managing the Daily call UI, participant views, audio monitoring, recording, and screen sharing. This is the largest and most complex file in the project.
- Native dependencies: `Daily` and `DailySystemBroadcast` (via CocoaPods, see `.podspec`)

**Android layer** (`android/src/main/java/com/smartwinnr/plugin/daily/`):
- `SmartWinnrDailyPlugin.java` — Capacitor bridge
- `SmartWinnrDaily.java` — Core logic
- `SmartWinnrDailyConfig.java` — Configuration

**Build output** (`dist/`):
- `esm/` — ES modules (from `tsc`)
- `plugin.js` — IIFE bundle (from Rollup, global name `capacitorSmartWinnrDaily`)
- `plugin.cjs.js` — CommonJS bundle

## Key Considerations

- iOS minimum deployment target is 13.0, Swift 5.1
- iOS 18 introduced stricter threading/memory enforcement — see `IOS18_CRASH_FIXES.md` for patterns to follow: always use `[weak self]` in async closures, dispatch UI updates to main thread, invalidate timers on creation thread, guard against double-dismiss
- The `onLeft` event listener is iOS-only currently
- API docs in README.md are auto-generated by `@capacitor/docgen` from JSDoc comments in `definitions.ts` — edit the source, not the README directly
- Publishing runs `prepublishOnly` which triggers a full build
